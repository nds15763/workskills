# PRD Writer · Manuel 的 PRD 风格

## 输出结构

**固定三层顺序：**

1. **模块清单** — 先列全部模块（编号 + 一句话说明），给读者整体感
2. **逐模块展开** — 每个模块分 **UI外显** / **交互逻辑** 两段写
3. **待确定项** — 汇总所有开放问题，统一放在最后

---

## Markdown 使用规范

### 1. 分序号 `1. 2. 3.`
用于：**模块/步骤有顺序或需要索引时**
- 模块清单（页面从上到下的层次）
- 分步骤的交互流程（如：点击 → 拉起面板 → 选择主播 → 点击进入）
- 状态按优先级排列时（如：未预约 / 已预约 / 已结束）

### 2. 分点 `-`
用于：**同类型的并列内容，归到同一个"点"下**

核心原则：**同类放在一起，不分散**。比如，同一模块下所有的"包含元素"归一个点，所有"状态"归一个点，不要每个状态各开一个独立子标题。

**✅ 对的写法：**
```
- **状态枚举**：
  - **未预约**：黑底白字「🔔 预约」，可点击
  - **已预约**：灰底灰字「✓ 已预约」，不可点击
  - **已结束**：灰底「📺 回放」，可点击
```

**❌ 错的写法（分散了）：**
```
**未预约**：黑底白字...
**已预约**：灰底灰字...
**已结束**：灰底...
```

其他"分点"例子：
- 一个模块下所有的"包含元素"归一个点列出来
- 一个区域的"内容来源"和"外显规则"分别各一个点
- 折叠相关内容（默认条数 + 展开按钮 + 可否收起）归一个点

### 3. 加粗 `**text**`
用于以下场景，保持一致：

| 场景 | 示例 |
|------|------|
| 模块名 / 区域名 | **顶部信息区**、**高光切片区** |
| 元素名（组件/字段） | **预约按钮**、**节目标题**、**底bar** |
| 小节 sub-header | **UI外显**、**交互逻辑**、**内容来源** |
| 状态名 | **未预约**、**已结束** |
| 关键交互词 | **可交互**、**核心布局** |
| 数字规格（强调时） | **2条**、**52px**、**9:16** |

不要对普通描述性句子加粗，只加在名词/关键词上。

### 4. 高亮 `` `code` ``
用于：**技术值、字段名、函数名、CSS属性**

示例：
- 后台字段名：`节目详情页-直播开始时间`
- 函数名：`history.back()`、`adaptSovImages()`
- CSS/JS 值：`object-fit: cover`、`scrollLeft === 0`
- 取值格式：`UID · XXXXXXXX`

### 5. 引用块 `>`
用于：**研发说明、技术实现细节、待确认的补充说明**

```
> 📌 研发说明：每局合照是独立横向 scroll container，监听 scrollend 判断
> scrollLeft 是否触边，触边且有同向 touch delta 时触发局切换。
```

### 6. 书名号 `「」`
用于：**所有 UI 文案、按钮 copy、toast 文案、标签文字**

示例：「🔔 预约」、「直播回放在路上了，再等等吧～」、「查看全部预约 (N)」

## 语言风格

- **中文为主**，技术词汇保留英文：badge、toast、deeplink、scroll、container、emoji、SDK、tab、hero、UID、API
- **动词精准**：点击→拉起/触发/跳转/切换；滑动→吸附/切换；展示→根据…判断
- **数字精确**：52px、9:16、900字重、1px、2条、最多4枚 — 不用"大约"
- **因果链简洁**：触发条件 → 系统响应，一句话搞定，不废话
- **待确定项**：行内标注 **（待确定）**，统一在末尾汇总

---

## 两段式写法模板

**每个模块都按这个结构写，不多不少：**

```
**模块名**

**UI外显**
- **包含元素**：元素A、元素B、元素C
- **视觉规格**：尺寸、颜色、字重等关键数字
- **状态枚举**：
  - **状态A**：外观描述
  - **状态B**：外观描述

**交互逻辑**
- **判断条件**：什么时候出现/显示什么
- **点击行为**：触发 → 响应
- **特殊状态**：边界/异常处理
```

纯展示模块简化为：**UI外显** 描述，**交互逻辑** 写"纯展示，无交互"即可，不拉长。

---

## 折叠展开模式（固定写法）

遇到"默认展示N条，超出折叠"时，固定这样写：

> 默认展示 **2条**，超出折叠，展开按钮显示剩余数量「展开 (N)」，可收起。若无内容，整个区域不展示。

---

## 空状态模式（固定写法）

```
**UI外显**：居中布局，[emoji]（低透明度）+ [粗体标题] + [浅灰说明文字]。

**交互逻辑**：页面加载时自动检测，无内容时同步隐藏对应标题/区块，替换为空状态展示。纯展示，无额外操作入口。
```
