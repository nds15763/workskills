# Three Rulers（三把尺 · 候选收敛裁决桌）

这是给**我（助手）**的流程规则。当一堆反馈或候选方向要收敛成"这一版做什么"时，
我用它把战场从"吵方案"搬到上游的"认痛点、排尺子"，再机械过筛——
让被砍的方案输给尺，而不是输给某个人。

源头：2026-06-12 反编译 67变 两份文档——《体验反馈和思考》（上游摊牌现场：15+5 条
反馈逐条"原话→洞察→穷举→三态裁决"）+《决策梳理》（下游收敛：三把尺把 13 个候选
砍到 5 个）。结合本仓库 decision-tripwire 的目的本位检查与 knowledge-card-qa 的
用户裁决权改造而成。

2026-06-12 用户校准两条：
① **单人模式是主场景**——用户一个人产生无数想法，底料是他甩来的对话记录；
"各执一词"主要是**不同时刻的同一个人**在冲突。
② **裁决跟尺走**——既不是"我只摆表不想"（用户原话："我需要你帮我去想"），
也不是"裸推荐结论"（2026-06-10 用户原话："太快了……整个决策我都没看到，
现在你就告诉我漏斗和结论了？"）。规则见 Stage 4。

---

## 一、触发 / 不触发

**触发**（任一命中）：

- 用户甩来对话记录 / 一堆想法，说"思路乱了 / 想法太多 / 帮我收敛 / 帮我想"（**主场景**）
- 反馈 / 测试反馈 / 评审意见（≥3 条）需要整理成行动决策
- 候选方向 ≥3 个，要收敛成一版范围（"13 个方向砍到几个"）
- 各执一词争不下，需要把暗尺摆上桌——多人之间，或**一个人不同时刻**之间
  （周二要做工具的他 vs 周五认了娱乐的他）
- 用户说"收敛一下 / 砍方案 / 定范围 / 整理反馈 / 出决策梳理"

**不触发（转介）**：

- 单点取舍（A vs B 选一次）、凭"好做/我喜欢"挑东西 → `decision-tripwire`（它管起跳点，本 skill 管批量收敛；它是哨兵，本 skill 是法庭）
- 问题本身还说不清 → `problem-statement-card`
- 结论已收敛、只差用户拍板表达 → `knowledge-card-qa`
- 候选优劣争议是价值/审美之争（"哪个更高级"）→ 先 `say-show-boundary` 剥层

---

## 二、流水线：摊牌 → 立尺 → 过筛 → 收口

### Stage 0 · 底料：原话留源，禁止先翻译

- 输入的标准形态是**用户直接甩对话记录 / 聊天流水 / 语音转写**——抽取由我做，
  不要求用户先整理。
- 每条原话保留**来源**：多人场景标谁说的（子腾：「……」）；
  单人场景标**哪次对话、哪天、什么语境**（06-10 营销讨论中的你：「……」）。
  证据层和解读层物理分开。
- **时刻冲突摆明**：单人场景里，当前想法与早先已锁的决定冲突时，
  把两个时刻的原话并排摆出，由**现在的用户**裁决哪个还算数——不替他默认新的赢。
- 记录样本量 N。**N < 10 时，后续所有"洞察"必须标注为假设，不是统计结论。**
  （67变案例：球球一个人说"没有 1v3 好玩"，正确处理是并行试验，不是结构性结论。）

### Stage 1 · 升维：原话 → 洞察（聚类，可审计）

- 多条原话聚成一条洞察，洞察**必须列出覆盖的反馈编号**（"反馈 #1, #7, #13"）。
- **未被任何洞察覆盖的反馈进"未归类"区单列**，不许静默丢弃。
- **同源归并**：新提案若只是已有梯子的更高档，归并不另立。
  （案例："Johnny 的故事王不是独立提议，而是 2-1 这条问题的极致解法"。）
- **独立复推 = 强证据**：参与者自发说出与已选方案同向的话，记为反向验证。
  （案例：球球主动提"骰子"，验证五档骰子方向。）

### Stage 2 · 立尺（用户动作，我不替定）

- 三类尺，每类至少一把：
  1. **红线尺**——这一版不可破坏什么（如：商业闭环 / 主屏聚焦）
  2. **资源尺**——本版预算多大（**必须给量级**：几人日 / 轻量增量 or 大改造；禁止"感觉太重"）
  3. **有效性尺**——必须解决哪个根本痛点 / 是否有更好替代
- **尺先于方案**：尺没立完，禁止开始比较任何候选。
- **尺的排序是产品决策**，归用户——用 `knowledge-card-qa` 卡片锁定排序，我只提候选排序。
  各执一词争不下时，先问每人两个问题把暗尺逼出来：
  「你在防的是哪种失败？」「你这个方案成立的前提是什么？」

### Stage 3 · 摊选项：穷举 + 强度梯子 + 座位矩阵

- 每条洞察下穷举候选方法论。**防伪穷举下界**：至少摆出比直觉答案
  **更轻一档**和**更重一档**各一个——只有两个选项的"穷举"打回。
- **二元争论强制改强度梯子**：「要不要 X」改写成 L0~Ln 旋钮（每层包含前一层），
  问题从"做不做"变成"红线从哪一档开始被碰到"，答案取**满足洞察的最低有效档**。
  （案例：透明化 L0 不透明 → L2 花字气泡 → L5 完全明牌，选 L2——
  "信息流通到能聊天即可，不抢主屏"。）
- **多座位场景**（PVP / 直播 / 双边市场 / 社区）补**座位 × 档位影响矩阵**：
  列 = 座位（玩家 / 观众 / 金主 / 平台……每个座位的"赢"用"让谁做到什么"写），
  行 = 档位，格 = 该档对该座位的收益/伤害。座位间的冲突对显式标记。
- **假互斥排雷**（同 decision-tripwire 伴随病①）：候选真的互斥，还是排序/并行题？
  （案例：1v3 vs 2v2 → 两条并行链路，不是选择题。）

### Stage 4 · 过筛：裁决跟尺走

三态仍是 **✅ 选定 / ❌ 不选 / ⚠️ 暂缓**，但裁决怎么填取决于尺锁没锁：

- **尺已锁的部分 → 我直接填裁决**。此时裁决是推导不是偏好，必须标
  "由哪把尺推出"（"❌ 被红线尺砍：碰主屏聚焦"）。用户可审计推导、可推翻尺，
  但不需要逐格重想。
- **尺未锁的部分 → 禁止裸裁决，给"条件裁决"**：每个候选写清
  **大吉条件 / 转凶条件**（"若本版要的是付费信号 → ✅；若要的是种子关系 → ❌"），
  并把真正的决策点**上移成"锁哪把尺"**——用户要锁的是前提，不是逐个格子。
  用户烦的从来不是结论，是**前提没摆出来的结论**。
- **❌ 必须标尺名**——没有尺名的 ❌ 无效。
- **⚠️ 必须带回看触发条件**（"验证完 L2+骰子效果后" / "下一版"），
  没有触发条件的暂缓 = 糊掉。
- 摊牌表**摆完才许出现任何裁决**——禁止表没摆完就给结论
  （提前坍缩开放世界，2026-06-10 "太快了" 病）。
- 停车场不是坟场，默认措辞：**"不是不好，是不在这一版的预算里"**，并标未来去向。

### Stage 5 · 收口：必要性测试 + 倒逼项 + 总表

- 留下的候选过 **remove-one 必要性测试**：拿掉任一个，本版一句话叙事崩不崩？
  不崩的降级为可选——必要性测试比价值排序严格，挡住"都挺好"的糊账。
- **倒逼项单独标来源**：不是反馈直接要求、而是为闭环一致性反推出来的追加
  （"叙事改了，结算也得跟着改"），与痛点直推项分开记。
- 产出**决策路径总表**（全部候选 + 状态，含被砍的）+ **带数字的一句话总结**
  （"13 砍到 5 = 3 个基础闭环 + 2 个倒逼"）。

## 三、反模式

- 没立尺就开始比方案（= decision-tripwire 物本位病的批量版）。
- **摊牌表没摆完就给结论**——提前坍缩开放世界（"太快了"病：用户要看见
  全部选项和前提之后再收，不是直接收到结论）。
- **尺没锁就给裸推荐**——推荐必须以"大吉/转凶条件"形态出现，前提先行。
- **只摆表不思考**——条件裁决、尺名、大吉/转凶前提是我的思考劳动，
  不许用"中立"当偷懒借口（用户原话："我需要你帮我去想"）。
- 洞察不带反馈编号——解读洗白成事实。
- "穷举"只摆两个选项就收工；梯子缺"更轻一档"。
- ❌ 不标尺名；⚠️ 不带回看触发条件。
- 把并行/排序题做成互斥单选。
- 单人场景里，新想法与旧锁定冲突时**默认新的赢**——必须摆出两个时刻原话让用户裁决。
- 替用户锁尺并直接开始执行。
- 把小样本（N<10）测试洞察当统计结论写进文档。
- 资源尺用"感觉太重 / 工程改造太大"裁决而不给量级——ROI 尺是最容易被滥用的一把，
  它能砍掉一切；不给数字的 ROI 裁决打回。

## 四、输出形态

```text
① 摊牌表（每条洞察一组）：
   # | 原话(留源) | 洞察(覆盖#) | 候选(梯子 L0~Ln) | 评价
     | 裁决: 尺已锁→推导结果+尺名 / 尺未锁→大吉条件/转凶条件
② （多座位场景）座位 × 档位影响矩阵
③ 决策路径总表 + 停车场（含回看触发条件）
④ 带数字的一句话总结
```

输出后**停下**，把"待锁的尺"做成 knowledge-card-qa 卡片（2-4 候选 + 证据），
等用户锁尺。尺一锁，对应的条件裁决自动坍缩成确定裁决，我再做 Stage 5 收口。

---

## 不做

- ❌ 不替用户**锁尺**——尺的取值和排序永远是用户的产品决策；
  尺锁定之后的推导裁决是我的工作，不再回头请示每个格子。
- ❌ 不处理单点取舍（转 `decision-tripwire`）。
- ❌ 不把卡片/表格当权威源——裁决落定后该进 OpenSpec/知识库的，走对应 skill 沉淀。
