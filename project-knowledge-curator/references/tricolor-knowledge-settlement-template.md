# 三色知识沉淀模板

```markdown
## PRD知识

### 白知识
- 已确认有效、可进入默认基线的产品目标、业务规则、用户流、验收口径

### 灰知识
- 有潜在复用价值，但尚未进入默认基线的 PRD 结论

### 黑知识
- 已明确弃用、隔离或不应继续进入执行链路的 PRD 内容

## 约束知识

### 白知识
- 已确认有效的门闩、流程限制、环境限制、禁止事项

### 灰知识
- 待确认的规则、限制或待验证约束

### 黑知识
- 已弃用或已被替代的约束

## 架构知识

### 白知识
- 已确认有效的模块边界、数据流、对象模型、控制面设计

### 灰知识
- 待进一步确认的架构方案或实现路径

### 黑知识
- 已废弃或已被替代的架构结论

## 影响的真源
- main-phase
- skills
- `<business_domain>/README.md`
- `<business_domain>/knowledge/README.md`
- code

## 待确认问题
- 仍需用户或主 Agent 决策的问题
```

规则：

- 不做逐轮对话转录
- 优先沉淀最终成立的结论
- 每条结论都必须先归类到 `prd | constraint | architecture` 之一
- 白知识只写已确认规则
- 灰知识写待确权项
- 黑知识写已弃用或已隔离内容
